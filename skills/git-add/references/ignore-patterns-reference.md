# Ignore patterns reference

Comprehensive catalog of file patterns that developers should typically
add to `.gitignore` rather than stage for commit.

---

## Pattern categories

### 1. Build artifacts

Build tools generate these files during compilation.

**Patterns**: `dist/`, `build/`, `out/`, `target/`, `*.o`, `*.so`,
`*.dll`, `*.exe`, `*.class`, `*.jar`, `*.pyc`, `*.pyo`, `*.wasm`,
`bin/`, `obj/`

**Framework-specific**:

- Node.js: `dist/`, `build/`, `lib/` (if compiled)
- Java: `target/`, `*.class`
- Python: `__pycache__/`, `*.pyc`, `*.pyo`, `*.pyd`
- Rust: `target/`
- C#/.NET: `bin/`, `obj/`, `*.dll`, `*.exe`

### 2. Dependencies

Package managers install these third-party packages.

**Patterns**: `node_modules/`, `vendor/`, `.pnpm/`, `__pypackages__/`,
`venv/`, `.venv/`, `env/`, `bower_components/`, `jspm_packages/`

**Package manager detection**:

- `package.json` exists → ignore `node_modules/`
- `Pipfile`/`requirements.txt` exists → ignore `venv/`, `.venv/`
- `composer.json` exists → ignore `vendor/`
- `Cargo.toml` exists → ignore `target/`

### 3. IDE/editor files

Development tools create these configuration and cache files.

**Patterns**: `.vscode/`, `.idea/`, `.vs/`, `*.swp`, `*.swo`, `*.swn`,
`*.sublime-project`, `*.sublime-workspace`, `.project`, `.classpath`,
`.settings/`, `*.iml`, `.atom/`, `*.code-workspace`

**Exception**: `.vscode/settings.json` or `.vscode/launch.json` may
function as project-specific `config` (use discretion).

### 4. Environment files (Sensitive - hard block)

Files that contain environment-specific settings or secrets.

<!-- vale Vale.Spelling = NO -->

**Patterns**: `.env`, `.env.local`, `.env.*.local`, `*.local`,
`config/local.*`, `config.local.*`

<!-- vale Vale.Spelling = YES -->

**Exception**: `.env.example`, `.env.template` serve legitimate purposes
(no secrets, just structure).

### 5. Logs

Development and runtime processes generate these log files.

**Patterns**: `*.log`, `logs/`, `log/`, `*.log.*`, `npm-debug.log*`,
`yarn-debug.log*`, `yarn-error.log*`, `lerna-debug.log*`, `.npm/`

### 6. Cache files

Tools and frameworks store temporary cache data here.

**Patterns**: `.cache/`, `cache/`, `*.cache`, `.pytest_cache/`,
`.ruff_cache/`, `__pycache__/`, `.eslintcache`, `.stylelintcache`,
`.parcel-cache/`, `.next/`, `.nuxt/`, `.turbo/`

### 7. Temporary files

Development and testing processes create these temporary files.

**Patterns**: `tmp/`, `temp/`, `.tmp/`, `*.tmp`, `*.temp`, `*.swp`,
`*~`, `.DS_Store`

### 8. Sensitive files (Hard block - never stage)

Files likely containing secrets, keys, or credentials.

**Patterns**: `*.key`, `*.pem`, `*.p12`, `*.pfx`, `secrets.*`,
`credentials.*`, `*.crt`, `*.cer`, `*_rsa`, `*_dsa`, `*_ecdsa`,
`*.keystore`, `*.jks`, `.aws/credentials`, `.ssh/id_*`

**Detection**: file extension/name contains `secret`, `credential`,
`password`, `private`, or path contains `.ssh/`, `.aws/`, `.gnupg/`

**Action**: **always** block, warn user, add to gitignore automatically.

### 9. Operating system files

Operating systems generate these files.

**Patterns**: `.DS_Store`, `Thumbs.db`, `ehthumbs.db`, `Desktop.ini`,
`$RECYCLE.BIN/`, `._*`, `.Spotlight-V100`, `.Trashes`

### 10. Generated code

Auto-generated files that users shouldn't manually edit.

**Detection markers**: file contains `DO NOT EDIT`, `@generated`,
`auto-generated`, `autogenerated`,
`This file is automatically generated` (in first 10 lines)

**Patterns**: `*.min.js`, `*.min.css`, `*.map` (outside `src/`)

**Exception**: `package-lock.json`, `yarn.lock` often INTENTIONALLY
committed for reproducible builds.

## Decision tree for non-versioned files

```text
Is file in node_modules/, vendor/, venv/, or similar?
├─ Yes → IGNORE (Category 2)
└─ No → Continue

Does file match sensitive patterns (*.key, *.pem, secrets.*, .env*)?
├─ Yes → HARD BLOCK + IGNORE (Category 8)
└─ No → Continue

Is file in dist/, build/, out/, target/?
├─ Yes → IGNORE (Category 1)
└─ No → Continue

Does file have .log extension or in logs/?
├─ Yes → IGNORE (Category 5)
└─ No → Continue

Is file .DS_Store, Thumbs.db, or similar OS file?
├─ Yes → IGNORE (Category 9)
└─ No → Continue

Is file *.tmp, *.temp, or in tmp/?
├─ Yes → IGNORE (Category 7)
└─ No → Continue

Is file in .cache/, __pycache__/, .next/, or similar?
├─ Yes → IGNORE (Category 6)
└─ No → Continue

Is file in .vscode/, .idea/, *.swp, or similar?
├─ Yes → IGNORE (Category 3, check exceptions)
└─ No → Continue

Does file contain "DO NOT EDIT" or "@generated" in first 10 lines?
├─ Yes → IGNORE (Category 10)
└─ No → Continue

Is file > 10 MB and not in assets/, public/, data/?
├─ Yes → CONSIDER IGNORING (warn user)
└─ No → Continue

File likely serves legitimate purpose → ANALYZE CONTENT for staging
```

## Exceptions

| Case                    | Criteria                                              | Decision                                                                                                                                                 |
| :---------------------- | :---------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Lock files**          | `package-lock.json`, `yarn.lock`, `Gemfile.lock`      | If project has committed lock file → commit changes<br>If new and no lock files exist → probably commit<br>If in `node_modules/` sub-dependency → ignore |
| **Source maps**         | `*.map` files                                         | In `dist/`, `build/` → ignore (generated)<br>In `src/` → might serve legitimate purpose<br>Alongside minified files → probably ignore                    |
| **Local configuration** | `config.local.js`, `settings.local.json`              | Ignore (personal overrides)<br>Exception: only structure, no values → might function as template                                                         |
| **Coverage reports**    | `coverage/`, `.nyc_output/`, `htmlcov/`, `*.coverage` | Ignore (generated)                                                                                                                                       |

## Framework-specific quick reference

| Framework      | Ignore patterns                                                                                                   |
| :------------- | :---------------------------------------------------------------------------------------------------------------- |
| **Node.js/JS** | `node_modules/`, `dist/`, `build/`, `.next/`, `.nuxt/`, `.cache/`, `*.log`, `npm-debug.log*`, `.eslintcache`      |
| **Python**     | `__pycache__/`, `*.py[cod]`, `*.so`, `venv/`, `.venv/`, `.pytest_cache/`, `.ruff_cache/`, `.coverage`, `htmlcov/` |
| **Java**       | `target/`, `*.class`, `*.jar`, `*.war`, `.settings/`, `.classpath`, `.project`                                    |
| **Rust**       | `target/`, `Cargo.lock` (for libraries)                                                                           |
| **Go**         | `*.exe`, `*.exe~`, `*.dll`, `*.so`, `*.dylib` (binary name varies by project)                                     |
| **Ruby**       | `vendor/bundle/`, `.bundle/`, `*.gem`, `.ruby-version` (sometimes committed)                                      |
| **PHP**        | `vendor/`, `composer.lock` (sometimes committed)                                                                  |

## Usage guidelines

1. **Check categories in order** (1-10) for fastest detection.

2. **Hard block sensitive files** (Category 8) immediately.

3. **Exercise caution**: When in doubt, ignore rather than accidentally
   stage.

4. **Check project context**: Look for existing `.gitignore` patterns.

5. **Consider size**: Scrutinize files > 10 MB.

6. **Warn on sensitive**: Always warn user when blocking sensitive
   files.

7. **Respect nested ignores**: Use closest `.gitignore` for patterns.

---

## Gitignore format reference

**Pattern syntax**:

- `file.txt`: ignore in any directory

- `dir/`: ignore this directory (trailing `/`)

- `*.log`: ignore all .log files

- `!important.log`: exception (don't ignore)

- `**/*.tmp`: ignore .tmp files in any subdirectory

- `/root-only.txt`: ignore only in repository root

**Best practices**: use trailing `/` for directories, specify patterns
when possible, group related patterns, and add comments with `#`.
